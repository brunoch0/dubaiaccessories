# Loyverse 무료 vs 애드온 비교 (파트너 설득용)

> 전략: 무료버전으로 시스템 가치를 먼저 증명 → **B2C 쇼핑몰 오픈 전**을 애드온 전환 데드라인으로.
> 애드온 = Integrations $9 + Advanced Inventory $29, 매장당 $38 × 2 = **$76/월 (~AED 280)**

## 비교표

| | 🆓 무료버전 (지금) | 💳 애드온 ($76/월) |
|---|---|---|
| 판매 데이터 수집 | 직원이 POS 화면 보고 **수기 전사 계속** (현재 분기 7,305행) | API 자동 수집 — **수기 전사 업무 소멸** |
| 재고 데이터 | 매일 CSV 다운로드 → 시스템 업로드 (직원 5분/일) | 실시간 자동 싱크 |
| 재고 최신성 | 하루 1회 | 실시간 |
| 파손/변색 차감 (Deduct) | 시스템에 수동 입력 | POS에서 사유 기록 → 자동 반영 (Stock Adjustment) |
| 품절/재주문 관리 | 업로드된 재고 기준 (하루 지연) | 실시간 저재고 알림 + 발주서 |
| 매장간 이동 | 수기 기록 | Transfer Order (WareHouse 포함) |
| 고객-영수증 매칭 (고객분석 시트) | 수기 유지 | 영수증 API → **반자동화 가능** |
| **온라인 쇼핑몰 재고 싱크** | 하루 1회 → **오버셀 위험** (품절 상품에 주문 유입) | 실시간 차감 → 오버셀 방지 |
| 월 비용 | $0 | $76 (~AED 280) |

## 핵심 설득 논리 (숫자)

- 판매 전사 + 재고 정리에 직원이 쓰는 시간: **하루 1~2시간 × 2매장 ≈ 월 60~120시간**
- UAE 매장 직원 인건비 기준 시간당 AED 15~25만 잡아도 **월 AED 900~3,000 상당의 업무**
- 애드온 AED 280/월 → **첫 달부터 3~10배 회수**. 게다가 전사 오류(마이너스 재고 42건의 원인 중 하나)도 사라짐
- 쇼핑몰 오픈 시점부터는 선택이 아니라 필수: 하루 1회 싱크로는 온라인 주문-매장 판매 충돌(오버셀)을 막을 수 없음

## 단계별 시나리오 (파트너에게 제시할 그림)

```
지금       : 무료버전 + 새 시스템 (CSV 업로드 방식) — 비용 0으로 시작
1~2개월 후 : 시스템 가치 확인되면 애드온 ON → 수기 업무 소멸 (업그레이드 스위치만)
쇼핑몰 오픈 : 애드온 필수 (실시간 재고 차감)
```

시스템은 처음부터 두 모드를 다 지원하게 설계 (CSV 업로드 ↔ API 자동) — 애드온 켜는 날 코드 변경 없이 전환.
