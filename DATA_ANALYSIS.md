## 3. 구분: POS 유료화로 해결 vs 빌드 필요

### 🟡 Loyverse 유료화로 사라지는 수기 업무 (월 $76: Advanced Inventory $29×2 + Integrations $9×2)

| 수기 시트 | 대체 기능 |
|-----------|-----------|
| 판매 품목 (7,305행 전사) | Integrations API 자동 수집 |
| 일별 매출 정산 | 백오피스 리포트 + API export |
| 재고현황 롤링 | Loyverse 실시간 재고 (이미 추적 중 — 시트가 중복 작업) |

| (창고 활용 시) 매장간 이동 | Transfer Order, WareHouse 로케이션 이미 등록됨 |

| 니즈 (수기 증거) | 빌드 대상 |

## 4. 시사점

1. 파트너의 수기 부담은 "재고"보다 **판매 전사(7,305행) + 고객 매칭(2,773행)**이 크다 — 전자는 POS 유료화가, 후자는 커스텀 빌드가 해결
2. 고객분석 시트의 존재 = 파트너가 이미 CRM을 원하고 있다는 뜻 (나이·국적·재방까지 추적) — B2C 쇼핑몰과 자연 연결되는 Bruno 핵심 가치 영역
3. 마이너스 재고 42건 → Phase 0 실사 정산의 필요성 입증
4. WareHouse 로케이션이 이미 있어 매장 확장/온라인 출고 구조 대비 완료
