## 2. SKU 규칙안 — 권장: 현행 유지 + 운영 지침만 변경

### 현행 구조 (10자리, 1,596개 준수)
```
25 0715 01 04
YY MMDD 카테고리 순번
```

### 권장안: 구조 변경 없음
1. **순번 01~99 사용** — 필드가 이미 2자리라 규칙 변경 없이 하루·카테고리당 99개까지 커버 (현재 관행 1~10은 운영 습관일 뿐)
2. **카테고리 코드 06~99 확장 예약** — 카테고리 추가 시 코드만 부여 (액세서리+α 확장 대비)
3. **구형 SKU는 시스템이 흡수** — 8자리 270개(연도 없음)·5자리 13개 등은 매핑 테이블로 처리. **기존 라벨 재작업 0건** (바코드는 13자리로 이미 통일돼 있음)
4. 신규 입고분부터 10자리 규칙 100% 준수 (시스템 등록 화면이 자동 채번해주므로 실수 원천 차단)

### 기각안: 전면 재채번
새 체계로 전 상품 재등록 — 라벨 ~2,500개 재작업 + 매장 영업 중단 리스크. **비추천.**

## 3. 정제 실행 순서 (합의 후)
1. 시스템에서 SKU 코드 기준 카테고리 자동 복원 (스크립트)
2. 예외(코드 06, Van2, 빈 값, SKU 비정형 23개) 수작업 분류 — 파트너와 30분
3. 정제된 마스터를 Loyverse에 역반영 (CSV import) → POS와 시스템 카테고리 일치
4. 마이너스 재고 42건 실사 정산 (매장별 반나절)
